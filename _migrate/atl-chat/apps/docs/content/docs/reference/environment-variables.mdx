Values are loaded from **`.env`** (copy from `.env.example`) and optional overlays **`.env.dev`** / **`.env.prod`**. The root script `scripts/prepare-config.sh` substitutes `${VAR}` into UnrealIRCd, Atheme, bridge, and The Lounge templates.

This page documents variables **used by this monorepo**. The external **ATL Portal** uses its own env (see that repo‚Äôs `docs/ENV_VARS.md`); bridge auth uses **`BRIDGE_PORTAL_TOKEN`** here and **`BRIDGE_SERVICE_TOKEN`** on the portal ‚Äî they must match when Portal identity is enabled.

## How environment files work

1. **`.env`** ‚Äî baseline; copy from `.env.example`.
2. **`.env.dev`** ‚Äî dev overlay for `just dev` (from `.env.dev.example`).
3. **`.env.prod`** ‚Äî production-only overrides for `just prod`.

```bash
cp .env.example .env
cp .env.dev.example .env.dev
just dev
```

Compose merges `.env` then the overlay. **`apps/docs`** deploy (Alchemy / Workers) uses separate vars ‚Äî see [Docs (deployment)](#docs-cloudflare-workers-via-alchemy) below; those are not in the root `.env.example`.

## TLS and IRC client behaviour (explicit flags)

These are set in `.env.example` so dev/prod behaviour does not depend on a single `ATL_ENVIRONMENT` variable. `scripts/prepare-config.sh` uses **`BRIDGE_IRC_TLS_VERIFY`** (if set) when exporting **`IRC_TLS_VERIFY`** for UnrealIRCd-related substitution.

| Variable | Description | Default (`.env.example`) |
|---|---|---|
| `IRC_TLS_VERIFY` | Verify TLS when connecting to IRC (e.g. bridge ‚Üí IRC) | `false` |
| `IRC_LOUNGE_REJECT_UNAUTHORIZED` | Node TLS `rejectUnauthorized` for The Lounge ‚Üí IRC | `false` |
| `IRC_WEBSOCKET_USE_TLS` | Enable TLS on UnrealIRCd listener in generated config | `true` |
| `BRIDGE_IRC_TLS_VERIFY` | Passed to the bridge container; prefer this for bridge-specific TLS | `false` |

## Web frontend (`apps/web`, Next.js)

| Variable | Description | Default (`.env.example`) |
|---|---|---|
| `NEXT_PUBLIC_IRC_WS_URL` | Browser ‚Üí IRC WebSocket | `wss://irc.localhost/ws` |
| `NEXT_PUBLIC_XMPP_BOSH_URL` | Browser ‚Üí XMPP BOSH (`/http-bind` on Prosody HTTP) | `https://xmpp.localhost:5281/http-bind` (TLS via `atl-xmpp-nginx`; use `http://localhost:5280/http-bind` for plain HTTP to Prosody, e.g. `just web dev`) |

**XMPP WebSocket (RFC 7395)** is served at path **`/xmpp-websocket`** on Prosody‚Äôs HTTP stack. With the dockerized nginx sidecar, use **`wss://<XMPP_DOMAIN>:5281/xmpp-websocket`** (or **`wss://‚Ä¶/ws`**, which nginx proxies to the same upstream). Do not confuse with BOSH at **`/http-bind`**.

`NEXT_PUBLIC_*` values are baked in at **build** time for the web app ‚Äî set them before `pnpm build` / image build for your domain.

Fluux‚Äôs default URLs use **port 443** (`https://<jid-domain>/.well-known/‚Ä¶`, **`wss://<jid-domain>/ws`**). **`atl-xmpp-nginx`** publishes **`127.0.0.1:443` ‚Üí :443** (same TLS as **5281**) so those defaults work. If you cannot bind **127.0.0.1:443**, drop that mapping and set **Server (optional)** to **`wss://<XMPP_DOMAIN>:5281/ws`** or **`wss://localhost:8443/ws`** (Fluux-side proxy after rebuild).

## Portal integration (external ATL Portal)

The **Portal** app reads these from **its own** `apps/portal/.env` (see Portal `docs/ENV_VARS.md`). They are **not** substituted by `prepare-config.sh` in atl.chat; document them here so operators can align URLs and secrets with this stack.

Typical values when Portal talks to **Docker service hostnames** (from Portal‚Äôs container network) vs **public hostnames** (from a host-run Portal) differ ‚Äî use the hostname Portal can actually reach.

| Variable | Role |
|---|---|
| `IRC_ATHEME_JSONRPC_URL` | Atheme JSON-RPC, e.g. `http://atl-irc-server:8081/jsonrpc` |
| `IRC_UNREAL_JSONRPC_URL` | UnrealIRCd JSON-RPC HTTPS URL |
| `IRC_UNREAL_RPC_USER` / `IRC_UNREAL_RPC_PASSWORD` | Match `WEBPANEL_RPC_*` |
| `PROSODY_REST_URL` | Prosody admin API base ‚Äî include path, e.g. `http://atl-xmpp-server:5280/admin_api` |
| `PROSODY_REST_TOKEN` | Bearer from `just prosody-token` (preferred) |
| `PROSODY_REST_USERNAME` / `PROSODY_REST_PASSWORD` | Legacy HTTP Basic for admin API ‚Äî avoid if Bearer is configured |
| `IRC_SERVER` / `IRC_PORT` | Client-style IRC connection target |

> **Warning:** Treat RPC and REST tokens as admin credentials; rotate defaults before production.
